# 2026-04-20 Refactor Pass — Upload Orchestration, Helpers Split, Route Thinning

High-impact, low-risk refactors that unify the TB upload path in the frontend,
decompose `backend/shared/helpers.py` into cohesive modules, and extract
mixed-concern logic out of the organization and billing route layers.

No behavior change. No API contract change. All refactors are additive
(new modules) + shim re-exports, so existing import sites keep compiling.

---

## Priority 1 — Frontend upload unification

### What was centralized

Previously, TB upload behavior was split across three call sites:

1. `hooks/useTrialBalanceUpload.ts` — built `FormData` inline, called `uploadFetch`,
   interpreted 401/403/EMAIL_NOT_VERIFIED response semantics inline.
2. `hooks/useAuditUpload.ts` — generic hook for other tools; uses `uploadFetch`
   via a `getActionableError` mapper.
3. `app/tools/multi-period/page.tsx` — bypassed both of the above and called
   `apiPost('/audit/trial-balance', token, formData)` directly, which went through
   the generic `apiPost` transport (CSRF-aware but no upload-specific error
   shaping, and no discriminated `EMAIL_NOT_VERIFIED` branch).

### What was extracted

New: **`frontend/src/utils/trialBalanceUpload.ts`**

* `TrialBalanceUploadArgs` — input shape (file, threshold, overrides, mapping,
  sheets, engagement, preflight token).
* `buildTrialBalanceFormData(args)` — single source of truth for the wire format.
* `uploadTrialBalance(args, token?)` — POSTs via `uploadFetch` and returns a
  discriminated `TrialBalanceUploadOutcome`:
  - `success` with `AuditResultResponse`
  - `audit_error` (backend `{status: "error"}` body)
  - `auth` (401)
  - `email_not_verified` (403 with EMAIL_NOT_VERIFIED code — takes precedence over 403)
  - `forbidden` (generic 403)
  - `transport` (422 / 429 / 5xx / network)
* `classifyTrialBalanceFailure(raw)` — pure mapper used inside `uploadTrialBalance`
  and testable in isolation.

Migrations:

* `app/tools/multi-period/page.tsx::auditFile` — now calls `uploadTrialBalance`.
  Gains the same auth / EMAIL_NOT_VERIFIED / 403 semantics as the main TB flow.
* `hooks/useTrialBalanceUpload.ts::runAudit` — delegates FormData construction
  and transport to `uploadTrialBalance`. Hook retains its state machine
  (progress indicator, recalc state, mapping-required short-circuit, activity log).

Tests: **`frontend/src/__tests__/trialBalanceUpload.test.ts`** — 15 tests covering
FormData construction (all optional fields, empty-omit rules), failure
classification (every branch), and the async outcome contract.

### Verification

* `npx jest --testPathPatterns=trialBalanceUpload` → 15 passed.
* `npx jest --testPathPatterns="useTrialBalanceUpload|useAuditUpload|uploadTransport"`
  → 20 passed (existing tests unaffected).
* `npx next build` → passes; all routes render as `ƒ (Dynamic)` (CSP
  nonce-based rendering contract intact).

## Priority 2 — Backend `shared/helpers.py` decomposition

The 1,075-line helpers module became the landing zone for anything that
touched multiple domains. Split into cohesive modules; `shared/helpers.py`
is now a ~150-line re-export shim so every existing `from shared.helpers
import X` call site continues to resolve with zero code change.

### New modules

| Module | Responsibility |
|---|---|
| `shared/upload_pipeline.py` | Ten-step upload defense: `validate_file_size`, XLSX archive inspection, magic-byte checks, text sniff, row-count estimates, format-specific parsers, `_validate_and_convert_df`, `parse_uploaded_file[_by_format]`, `memory_cleanup`. |
| `shared/filenames.py` | `hash_filename`, `get_filename_display`, `safe_download_filename`, `sanitize_csv_value`, `escape_like_wildcards`. |
| `shared/background_email.py` | `safe_background_email` — BackgroundTasks wrapper for SendGrid calls with structured-log failure recording. |
| `shared/tool_run_recorder.py` | `maybe_record_tool_run` + `_log_tool_activity` — engagement ToolRun + unified ToolActivity feed persistence (no transactional boundary; caller commits). |

### What remains in `shared/helpers.py`

* Re-exports from the 4 new modules (back-compat).
* Small enum-coercion helpers (`try_parse_risk`, `try_parse_risk_band`).
* Lightweight JSON form parsers (`parse_json_list`, `parse_json_mapping`).
* Client access policy (`is_authorized_for_client`, `get_accessible_client`,
  `require_client`) — these remain co-located because they depend on `User`,
  `Client`, `OrganizationMember`, and `require_current_user`, and aren't
  consumed by enough other call sites to justify their own module today.

## Priority 3 — Route thinning (organization + billing)

### What was extracted

New: **`backend/shared/organization_policy.py`**

Moved from `routes/organization.py`:

* `slugify(name)` — URL-safe slug (≤90 chars)
* `unique_slug(db, base)` — collision-avoiding slug generation
* `get_user_org(db, user)` — 404 if not an org member
* `require_admin(db, user, org)` — 403 unless OWNER or ADMIN
* `require_owner(db, user, org)` — 403 unless OWNER
* `apply_org_subscription_to_user(db, user, org)` — new named helper that
  replaces ~25 lines of inline logic in `accept_invite`; handles the fast
  path (org.subscription_id resolves) *and* the owner-backfill fallback
  (looks up the owner's ACTIVE/TRIALING subscription and writes
  `org.subscription_id` so subsequent accepts hit the fast path).
* `revert_user_tier_after_removal(db, user)` — new named helper that replaces
  ~18 lines of inline logic in `remove_member`; restores a user's personal
  subscription tier when they leave an org, else drops to FREE.

New: **`backend/shared/billing_access.py`**

Moved from `routes/billing.py::get_weekly_review_endpoint`:

* `require_billing_analytics_access(db, user)` — centralizes the weekly-review
  security gate (org member must be OWNER/ADMIN; solo user must own an active
  subscription). Route handler now one line.

### Routes became thinner

* `routes/organization.py`: net delta −60 lines. Private `_get_user_org`,
  `_require_admin`, `_require_owner`, `_slugify`, `_unique_slug` eliminated;
  invite-accept + member-remove tier plumbing collapsed to 1–2 lines each.
* `routes/billing.py`: net delta −30 lines in the weekly-review endpoint.

### Deferred: `routes/auth_routes.py`

Intentionally left untouched. The cookie/CSRF/refresh primitives are
security-sensitive and already well-factored (`_set_refresh_cookie`,
`_set_access_cookie`, `_clear_*_cookie`). No net-positive extraction
identified under the "moving code, not inventing abstractions" guidance.

## Risk notes

* **Re-export shim surface.** A `from shared.helpers import X` call-site
  audit covered 50+ files. Every symbol that previously existed still
  resolves via the shim. A test asserts this explicitly.
* **Upload behavior preservation.** The frontend multi-period page now
  inherits the EMAIL_NOT_VERIFIED / 403 / 401 discrimination that the main
  TB page had. Worst case: a user whose previous `apiPost` silently swallowed
  an EMAIL_NOT_VERIFIED response now sees an actionable error message.
  This is net-positive but is a user-visible message change on a previously
  untestable edge case.
* **Auth/security/zero-storage untouched.** No changes to JWT, cookie,
  CSRF, rate-limit, or entitlement code paths. The billing weekly-review
  ACL is functionally identical (same HTTP status codes, same messages, same
  branch order) — only the location changed.

## Deferred items

| Item | Reason |
|---|---|
| `routes/auth_routes.py` extraction | Already reasonably thin; cookie/token primitives are security-critical and touching them without a specific bug or audit finding is net-negative. |
| `routes/billing.py::stripe_webhook` further split | Sprint 676 already reserved webhook-handler unit tests as a dedicated coverage sprint; extracting the signature-verify / dedup / error-mapping triad is better paired with that coverage work. |
| `useTrialBalanceUpload` full decomposition into composable hooks | The hook's state machine (progress indicator, recalc debounce, mapping-required handoff) is tightly coupled to consumer semantics. Not a drop-in extraction; deserves a focused sprint with Playwright coverage of the mapping-required flow before any split. |
| `shared/helpers.py` client-access helpers move | `is_authorized_for_client` / `get_accessible_client` / `require_client` still live in the shim. Small enough and specific enough that a dedicated module wasn't justified under the "prefer moving code, avoid new abstractions" directive. |
| `shared/entitlement_checks.py` wiring audit (Sprint 678) | Out of scope — separate Post-Audit Remediation Batch sprint. |
