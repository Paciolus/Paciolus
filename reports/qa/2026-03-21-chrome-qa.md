**Blocking issue found during setup:** The `users` table contained a stale `ORGANIZATION` tier value (from a prior schema era) which caused `500 Internal Server Error` on login. Fixed by updating to `FREE`. This would affect any database that was not migrated through the tier rename.

## 6. Console & Network Audit

### Console Errors Summary

| Error | Count | Severity | Component |
|-------|-------|----------|-----------|
| Hydration mismatch (ParallaxSection translateY) | 2 | Low | HomePage → HeroScrollSection |
| Hydration mismatch (sonification button) | 4 | Low | ToolsLayout → ToolLinkToast |
| `value.toFixed is not a function` | 3 | **High** | LeadSheetCard → formatCurrency |

**Root cause of `toFixed` error:** Backend returns `net_balance` as string type (`"45000.0"`) instead of numeric. The `formatCurrency` function in `LeadSheetCard.tsx` calls `.toFixed()` on the raw value. **Fix applied** during this QA session (coerce via `Number()`).

## 7. New Issues Discovered (Prioritized)

### P0 — Blocking

| # | Issue | Location | Details |
|---|-------|----------|---------|
| NEW-001 | `ORGANIZATION` tier enum crash | `models.py` / DB | Stale DB values cause 500 on login. Need migration to clean up or add backwards-compat enum value. |
| NEW-002 | `uploads_used_current_period` column missing | `subscriptions` table | Migration didn't add column. Causes 500 on TB upload. Required `ALTER TABLE` manual fix. |
| NEW-003 | `severity: "informational"` not in response schema | `diagnostic_response_schemas.py:572` | Backend engine returns `"informational"` severity, but Pydantic Literal only allows `high\|medium\|low`. Causes `ResponseValidationError` (500). **Fix applied** during QA. |

### P1 — High

| # | Issue | Location | Details |
|---|-------|----------|---------|
| NEW-004 | `value.toFixed is not a function` in LeadSheetCard | `LeadSheetCard.tsx:35` | Backend serializes `net_balance`, `total_debit`, `total_credit` as strings (via Decimal→str JSON). Frontend assumes numeric. **Fix applied** during QA. |
| NEW-005 | `GET /activity/recent?limit=5` returns 404 | Backend routes | Dashboard calls this on load but endpoint doesn't exist. Activity log writes succeed but reads fail. |
| NEW-006 | Lead sheet grouping shows 1 account per sheet | TB diagnostic response | 102-account TB produces 7 lead sheets each showing "1 Account" — expected A-Z grouping with multiple accounts per sheet. Likely a serialization or aggregation issue. |

### P2 — Medium

| # | Issue | Location | Details |
|---|-------|----------|---------|
| NEW-007 | Hydration mismatch: sonification button | `ToolLinkToast` | SSR renders button differently than client. Wrap in `useEffect` or `suppressHydrationWarning`. |
| NEW-008 | Hydration mismatch: ParallaxSection translateY | `HeroScrollSection` | Scroll-dependent values differ server/client. Use `suppressHydrationWarning` on motion divs. |
| NEW-009 | Classification quality flags contra accounts as anomalies | `classification_rules.py` | Allowance for Doubtful Accounts, Accumulated Depreciation, etc. flagged as "Asset with credit balance." These are normal contra accounts — should be excluded from sign_anomaly checks. |

### P3 — Low

| # | Issue | Location | Details |
|---|-------|----------|---------|
| NEW-010 | Next.js "2 Issues" dev overlay | Dev mode only | Shows hydration warnings; not visible in production build. |
| NEW-011 | Backend startup warns about missing Professional/Enterprise Stripe price IDs | `billing.price_config` | Professional monthly/annual checkout will fail. Non-blocking for Solo tier. |

## 8. Fixes Applied During QA

| Fix | File | Change |
|-----|------|--------|
| Add `"informational"` to severity Literal | `backend/shared/diagnostic_response_schemas.py:572,622` | `Literal["high", "medium", "low"]` → `Literal["high", "medium", "low", "informational"]` |
| Coerce `formatCurrency` input to number | `frontend/src/components/leadSheet/LeadSheetCard.tsx:35` | Added `Number(value) \|\| 0` coercion before `.toFixed()` |
| Fix stale `ORGANIZATION` tier in DB | `users` table | `UPDATE users SET tier = 'FREE' WHERE tier = 'ORGANIZATION'` |
| Add missing `uploads_used_current_period` column | `subscriptions` table | `ALTER TABLE subscriptions ADD COLUMN uploads_used_current_period INTEGER NOT NULL DEFAULT 0` |
| Point frontend to local API | `frontend/.env.local` | `NEXT_PUBLIC_API_URL=http://localhost:8000` |
